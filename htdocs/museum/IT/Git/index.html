Git入門：一人でもチームでも使えるバージョン管理の基本

Gitは、プログラムのソースコードなどの変更履歴を記録・追跡するためのバージョン管理システムである。

一人での開発効率を上げるだけでなく、チームでの共同作業に不可欠なツールだ。

この記事では、Gitの基本的な使い方から、GitHubと連携した実践的な開発フローまでを解説する。

第1章: Gitの初期設定と基本の3ステップ

最初にPCでGitを使うための準備を行う。この設定は一度だけ実施すればよい。

1-1. ユーザー情報の設定

コミット履歴に「誰が変更したか」を記録するため、名前とメールアドレスを設定する。


# グローバル設定（このPC上のすべてのGitリポジトリで使われる）
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

【補足】グローバル設定とローカル設定

--global はPC全体の設定。特定のプロジェクト（フォルダ）だけ違う名前を使いたい場合、そのフォルダで --global なしでコマンドを実行する。ローカル設定が優先される。

1-2. リポジトリの作成

バージョン管理を始めたいフォルダに移動し、次のコマンドを実行する。


# "my-project" フォルダに移動
cd my-project

# このフォルダをGitリポジトリとして初期化する
git init

これを実行すると、フォルダ内に .git という隠しフォルダが作成される。ここに変更履歴などの全情報が保存される。

1-3. 基本の3ステップ：「add → commit」

Gitでの作業は、基本的に以下の3ステップの繰り返しになる。

  1. 作業: ファイルを作成・編集する。
  2. ステージング (git add): 変更したファイルの中から、履歴に記録したいものを選ぶ。
  3. コミット (git commit): 選んだファイルの変更内容を、メッセージ付きで履歴に記録する。

# 現在のディレクトリ以下のすべての変更をステージングエリアに追加
git add .

# ステージングされた内容を "最初のコミット" というメッセージ付きで履歴に記録
git commit -m "最初のコミット"

【用語解説】

  • ワーキングディレクトリ: 現在作業しているファイルが存在するフォルダ。
  • ステージングエリア（インデックス）: コミット対象の変更を一時的に置く場所。「次のコミットに含める変更リスト」のようなもの。
  • リポジトリ: .git フォルダのこと。ファイルの全変更履歴が保存されている場所。
